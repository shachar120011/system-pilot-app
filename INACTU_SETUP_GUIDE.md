Inactu - מדריך הקמה ללקוח חדש (White-Label Setup)
מסמך זה מרכז את כל הפעולות הנדרשות כדי לשכפל את פלטפורמת Inactu ולהתאים אותה ללקוח חדש, מאפס ועד לאוויר (Production).

שלב 1: שכפול הפרויקט (GitHub)
היכנס למאגר (Repository) של Inactu ב-GitHub.

לחץ על Use this template (אם הוגדר כתבנית) או בצע Fork / Duplicate לפרויקט.

תן לפרויקט החדש שם מזהה (למשל: inactu-strauss).

שלב 2: הגדרות מותג ולקוח (קוד)
זהו השלב החשוב ביותר. כאן אנו משנים את חזות המערכת.

פתח את הקובץ src/config.ts.

עדכן את המשתנים הבאים בהתאם ללקוח:

clientName: שם הארגון (למשל: "קבוצת שטראוס").

clientSystemName: שם המערכת המוטמעת.

botWelcomeTitle / botWelcomeSubtitle: משפטי הפתיחה של הבוט.

פתח את הקובץ index.html (בתיקייה הראשית):

שנה את התגית <title> לשם הלקוח או המערכת (למשל: <title>Inactu - Strauss</title>).

(אופציונלי) החלף את קובץ ה-favicon.ico בלוגו של הלקוח.

פתח את הקובץ package.json ושנה את שדה ה-"name" לשם הפרויקט באנגלית (ללא רווחים).

שלב 3: הקמת דאטה-בייס נפרד (Google Sheets)
כדי שהנתונים של הלקוח החדש לא יתערבבו עם לקוחות אחרים, חובה ליצור שרת נתונים חדש.

היכנס ל-Google Drive וצור קובץ Google Sheets חדש.

פתח 3 גיליונות וקרא להם בדיוק כך:

Bug_Reports

Chat_Logs

Knowledge_Base

בכל גיליון, צור את שורת הכותרות הרלוונטית (תאריך, שם משתמש, מחלקה וכו').

לחץ על הרחבות -> Apps Script והדבק שם את קוד השרת (הכולל את פונקציות doPost ו-doGet של Inactu).

לחץ על Deploy -> New Deployment, בחר בסוג "Web App", והגדר שכל אחד ("Anyone") יכול לגשת.

העתק את ה-URL החדש שגוגל סיפק לך.

שלב 4: חיבור הקוד לדאטה-בייס החדש
פתח את הקובץ src/pages/AdminDashboard.tsx.

חפש למעלה את המשתנה APPS_SCRIPT_URL והחלף את הקישור שם בקישור החדש שהעתקת בשלב הקודם.

פתח את הקובץ src/pages/UserDashboard.tsx וחזור על אותה פעולה בדיוק (החלפת הקישור של APPS_SCRIPT_URL).
(טיפ לעתיד: כדאי להעביר את המשתנה הזה לתוך config.ts כדי לשנות אותו רק פעם אחת).

שלב 5: העלאה לאוויר (Vercel)
היכנס לחשבון ה-Vercel שלך ולחץ על Add New -> Project.

ייבא את המאגר החדש שיצרת ב-GitHub (inactu-strauss).

קריטי: לפני שאתה לוחץ על Deploy, פתח את ההגדרות של Environment Variables.

הוסף את המשתנה הבא:

Name: VITE_GEMINI_API_KEY

Value: [הדבק כאן את מפתח ה-API שלך מ-Google AI Studio]

לחץ על Deploy.

שלב 6: בדיקות וניקוי (QA)
היכנס לכתובת האתר ש-Vercel ייצר.

הזן קריאת טסט בצ'אט וודא שהבוט עונה (מוודא שה-API של ג'מיני תקין).

פתח תקלה יזומה ממסך "דיווח תקלה".

היכנס ללוח הבקרה למנהלים וודא שהתקלה והשאלה מופיעות שם.

היכנס ל-Google Sheets של הלקוח וודא שהנתונים נכתבו לשם בהצלחה.

✅ המערכת מוכנה למסירה ללקוח!
